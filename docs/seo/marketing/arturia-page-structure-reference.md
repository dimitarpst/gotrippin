# Reference page structure — Arturia MiniFuse (marketing anatomy)

**URL:** [MiniFuse 2 OTG — Overview](https://www.arturia.com/products/audio/minifuse-2-otg/overview)  
**Why look at it:** Hardware ≠ trip software, but the **information architecture** is a strong template for a **single product page** that sells one thing deeply.

## Section stack (abstracted)

1. **Hero** — Product name + **short category line** (their “Flexible USB audio interface”) + **one dense paragraph** (who it’s for, main benefit, proof hooks) + **primary CTA** + secondary (“find a dealer”).
2. **First belief section** — Big headline (“Record without limits”) + **subhead** + short supporting copy + **audience columns** (“For streamers”, “For podcasters”, …) each with **one tight paragraph** — _parallel use cases, same product_.
3. **Capability bands** — Repeated rhythm: **large two-word headline** + **H3** + body + optional **spec callouts** (their “2x Inputs”, “Loopback”, …). Several stacked blocks.
4. **“What’s your use-case?”** — Interactive or tabbed **modes** (Two-Way / Studio / Stream / …) with **one deep explanation per mode** — _maps to “planning solo vs group vs …” for us later if useful_.
5. **“Streaming made simple”** — Another headline + **annotated product** (hotspots on device). _For us: annotated screenshot or real UI map (when honest visuals exist)._
6. **Included / ecosystem** — “Best software included” + **grid of named items** with one line each. _Analog: integrations, export, mobile, AI — whatever is true._
7. **Trust** — Warranty / reliability. _Analog: privacy stance, data, “built for real trips”, testimonials later._
8. **Technical depth** — Collapsible or linked specs. _Analog: FAQ, “how sync works”, account/auth — keep truthful._
9. **Getting started** — Numbered steps (01–04) + links. _Analog: sign up → create trip → invite → map._
10. **Footer** — Legal, privacy, contact. _We still need real `/privacy` + `/terms`._

## What to steal for gotrippin

| Arturia pattern           | gotrippin direction                                                  |
| ------------------------- | -------------------------------------------------------------------- |
| One hero, one product     | **Single homepage** as the SEO story until more URLs ship            |
| Category line under title | Plain language (trip planner, group trips, map) — not jargon         |
| “For X / For Y” columns   | **Personas or trip types** (friends, couples, multi-city)            |
| Repeated section rhythm   | **Consistent blocks**: map, AI, sharing — each with headline + proof |
| Annotated hardware        | **Real UI** or simplified wireframe — no fake destination cards      |
| Use-case tabs             | Optional later: solo vs group vs “one driver, many opinions”         |
| Steps 01–04               | **Onboarding story** on the page (even if CTA goes to app)           |

## What not to copy

- Spec tables before the product is understandable.
- Claim density that outruns what the app actually does today.
- Visual language of audio gear (obviously).
