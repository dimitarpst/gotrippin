Introduction

Building a sleek, interactive color picker can add a touch of elegance to your web application. In this tutorial, we’ll walk through creating a dynamic color picker using Next.js and Framer Motion. Whether you’re familiar with Next.js or prefer using React.js, this guide will help you set up your project, organize your folder structure, and create reusable, animated components step-by-step.

We’ll start by setting up our development environment and installing essential packages like Framer Motion for animations and Prettier for code formatting. As we progress, we’ll create and animate various components, such as buttons, color boxes, and tabs, to ensure a seamless and visually appealing user experience. By the end of this tutorial, you’ll have a fully functional, customizable color picker component that you can easily integrate into your web projects. Let’s dive in and start building!

Let’s breakdown what this codes do.

  1. <AnimatePresence mode="wait">: This is a Framer Motion component. It helps manage the animations when components enter or leave the screen. The "wait" mode means it'll wait for the exiting component to finish its animation before showing the new one.
  2. Inside, we have two conditional renders. One for when selectedTab === 'Preset'. Another for when selectedTab === 'Custom'

3. Each condition wraps its content (PresetView or CustomView) in a <motion.div>. This div is what actually animates.

4. The animation properties:

  • initial: When the view first appears, it starts slightly to the right (x: 10) and invisible (opacity: 0)
  • animate: It then moves to its normal position (x: 0) and becomes fully visible (opacity: 1)
  • exit: When it's leaving, it slides slightly left (x: -10) and fades out (opacity: 0)
  • transition: This sets how the animation behaves. It takes 0.2 seconds. It uses a spring-type motion. It has a slight bounce (0.3)

5. key: This helps React and Framer Motion keep track of which view is which during animations.

In simple terms, when you switch tabs, the current view will slide out to the left and fade away, while the new view slides in from the right and fades in. This creates a smooth, pleasant transition between the Preset and Custom views.
